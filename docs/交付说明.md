## 4. 核心 API

工程层只需从 `index.ts` 导入:

```typescript
import {
  // 照片 → V-A
  photoToVA, fuseFeaturesToVA,
  // GPS 融合
  fuseVisualAndGPS, buildGPSReverseResult, resolveAdaptiveWeights,
  // 音乐 → V-A
  songToVA, getSongVA, spotifyFeaturesToVA, heuristicEstimateVA,
  // 匹配评分
  calcMatchScore, filterCandidatePool, calcExtendSimilarity,
  // 推荐
  recommend,
  // 用户偏好
  initUserPreference, processInteraction, updateGenreWeights,
  // 工具
  calcVADistance, findNearestEmotionLabel, resolveEmotionLabels,
  // 类型
  type Song, type PhotoFeatures, type UserPreference, type RecommendationResult,
} from './src/algorithm/index.js';
```

### 4.1 主流程 API

#### `photoToVA(features: PhotoFeatures): VAWithConfidence`
照片 8 特征 → V-A 坐标(纯视觉,不含 GPS)。

#### `fuseVisualAndGPS(input: FusionInput): FusionOutput`
视觉 V-A + GPS 反查结果 → 融合 V-A(含深夜修正、低置信度标记)。

#### `recommend(input: RecommendInput): RecommendationResult`
**推荐主入口**。输入照片情绪 + 场景 + 用户偏好 + 参考歌 + 全库,输出 15-20 首推荐。

#### `initUserPreference(answers, referenceSongs): UserPreference`
首次问卷 → 初始偏好模型。

#### `processInteraction(current, event): UserPreference`
处理一次用户交互,返回更新后的偏好(EMA 学习 + 中心漂移)。

### 4.2 辅助 API

| 函数 | 说明 |
|------|------|
| `buildGPSReverseResult(coord, poi, provider)` | POI → 场景+V-A 映射 |
| `songToVA(input)` | 歌曲 → V-A(多源融合) |
| `calcMatchScore(song, ctx)` | 单首歌完整评分 |
| `filterCandidatePool(songs, photoVA)` | 候选池过滤 |
| `findNearestEmotionLabel(va)` | V-A → 最近情绪标签 |
| `resolveEmotionLabels(va)` | 混合情绪判定 |
| `calcVADistance(a, b)` | V-A 加权欧氏距离 |

## 6. 工程层对接指南

算法层已封装完整,工程层需实现以下**注入点**:

### 6.1 视觉模型(必须)

实现照片 8 特征提取,输出 `PhotoFeatures`:
- F1 主色调 hue(0-360°, tone, confidence)
- F2 亮度(0-1, level, confidence)
- F3 饱和度(0-1, level, confidence)
- F4 场景(7 类, confidence)
- F5 时段(5 类, confidence)
- F6 天气(5 类, confidence)
- F7 人物(count, dominantEmotion, confidence)
- F8 构图(4 类, confidence)
- overallConfidence

### 6.2 GPS 反查(可选,推荐)

实现 `GPSReverseFunction`,调用高德/腾讯/OSM API,返回 `GPSReverseResult`:

```typescript
const gpsResult = buildGPSReverseResult(
  coordinate,
  poi,        // { name, typeCode, category, district? }
  'amap',     // provider
);
```

### 6.3 音乐库构建

构造 `Song[]` 数组,每首歌需含:
- `songId` / `title` / `artist`
- `layer`: hot / emotion / fallback
- `va`: V-A 坐标(用 `songToVA` 标注或人工标注)
- `genres`: 风格标签(用 `normalizeGenres` 归一化)
- `sceneTags`: 场景标签
- `language` / `hotRecency` / `decade`
- 可选: `neteaseTags` / `spotifyFeatures`

### 6.4 音频回归模型(可选)

若使用方法 A(最高精度),实现 `AudioRegressionFunction`:
- librosa 提取音频特征
- 预训练 XGBoost 模型预测 V/A

### 6.5 前端交互

`RecommendationResult` 包含:
- `lowConfidence`: 置信度低标记(前端可加轻交互)
- `isMixedEmotion`: 混合情绪标记
- `coreTracks` / `extendedTracks`: 推荐歌曲(带评分明细)

用户交互后调用 `processInteraction` 更新偏好。

## 8. 调参指南

所有可调参数在 [config/thresholds.ts](../src/algorithm/config/thresholds.ts),**调参只需改此文件**。

### 8.1 常见调参场景

| 目标 | 调整参数 |
|------|----------|
| 提升热歌占比 | ↑ CORE_MIN_HOT_COUNT, ↑ MATCH_WEIGHTS.scoreHot |
| 降低推荐雷同 | ↓ CORE_MAX_SAME_LABEL, ↓ EXTEND_SAME_SOURCE_GENRE_JACCARD_MAX |
| 放宽候选池 | ↑ CANDIDATE_POOL_VA_DISTANCE |
| 加强用户偏好 | ↑ MATCH_WEIGHTS.scorePref |
| 冷启动更保守 | ↑ COLD_START_INTERACTIONS, ↓ COLD_START_PREF_WEIGHT |
| 探索更激进 | ↑ EXPLORE_EPSILON |

### 8.2 权重调整约束

调整任何权重组后,确保**总和 = 1.0**:
- `MATCH_WEIGHTS`(6 维,和=1.0)
- 冷启动权重(和=1.0)
- `FEATURE_WEIGHTS`(8 特征,和=1.0)
- `calcScorePref` 内部权重(和=1.0)
- `calcExtendSimilarity` 内部权重(和=1.0)

> 例外:`HEURISTIC_V_WEIGHTS` 是加性位移模型(和=1.25),非凸组合,无需总和=1.0。

## 9. 设计决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 情绪模型 | V-A 二维 + 16 标签 | 平衡精度与可计算性 |
| V-A 距离 | 加权欧氏(V:0.6, A:0.4) | V 不匹配比 A 不匹配更难受 |
| 照片融合 | 8 特征加权 + 方向系数 | V 偏色彩,A 偏饱和度/人物 |
| GPS 策略 | 自适应权重(4 策略) | 视觉置信度驱动 |
| 音乐标注 | 3 层方法(A/B/C) | 精度递减,兜底覆盖 |
| 匹配策略 | 加性 base + 乘性 boost | 可解释 + 灵活 |
| 推荐 | 两阶段(8 核心 + 扩展) | 兼顾精度与多样性 |
| 偏好学习 | EMA + 中心漂移 | 无感建模,渐进收敛 |
| 阈值管理 | 集中在 thresholds.ts | 零 magic number,调参一处 |
